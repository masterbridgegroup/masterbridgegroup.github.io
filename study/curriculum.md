## 파일 규칙 (매일 자동 생성 시 지켜야 할 것)

1. 파일 이름: `dayNN.html` (예: day02.html, day15.html) — 이 폴더(study/)에 저장
2. 페이지 구성: day01.html과 같은 틀 사용
   - 상단: Day 번호 / 날짜 / 과목명 / 오늘의 주제
   - 본문: 개념 4~5개 (초보자용 쉬운 설명 + 비유 + 시험 포인트 박스)
   - 하단: 객관식 퀴즈 10문제 (클릭하면 바로 정답/해설, 마지막에 점수)
3. index.html의 `const days = [...]` 배열 맨 뒤에 새 Day 한 줄 추가
4. 어제 퀴즈에서 다뤘던 개념 1개를 "복습 한 스푼"으로 서두에 짧게 재등장
5. 전문용어는 반드시 괄호로 쉬운 설명 병기 (예: 프로토타입(시제품))

## 과목별 주간 주제

### 1과목 소프트웨어 설계 (Day 1~35)
- 1주: 소프트웨어 생명주기, 개발 방법론(폭포수·나선형·프로토타입·애자일·스크럼·XP), 현행 시스템 분석
- 2주: 요구사항 개발(도출→분석→명세→확인), 요구사항 분석 기법, UML 개요와 다이어그램 종류
- 3주: 유스케이스, 클래스 다이어그램, 시퀀스 다이어그램, 애플리케이션 설계(모듈화·결합도·응집도)
- 4주: 공통 모듈, 코드 설계, 디자인 패턴(생성·구조·행위), 객체지향 개념(캡슐화·상속·다형성)
- 5주: 인터페이스 요구사항, 인터페이스 대상 식별, 미들웨어, 1과목 총복습 + 모의 20문제

### 2과목 소프트웨어 개발 (Day 36~70)
- 1주: 자료구조(리스트·스택·큐·트리·그래프), 정렬 알고리즘 기초
- 2주: 이진 탐색, 해싱, 데이터베이스 개요, 통합 구현(단위 모듈)
- 3주: 형상 관리, 버전 관리 도구, 빌드 자동화, 제품 소프트웨어 패키징
- 4주: 애플리케이션 테스트(블랙박스·화이트박스), 테스트 레벨, 결함 관리
- 5주: 인터페이스 구현(JSON·XML·AJAX), 인터페이스 보안, 2과목 총복습 + 모의 20문제

### 3과목 데이터베이스 구축 (Day 71~105)
- 1주: 데이터베이스 기본 개념, DBMS, 스키마, 관계형 모델(릴레이션·키 종류)
- 2주: SQL 기초(SELECT·INSERT·UPDATE·DELETE), WHERE·ORDER BY·GROUP BY
- 3주: JOIN, 서브쿼리, DDL(CREATE·ALTER·DROP), DCL(GRANT·REVOKE), 뷰·인덱스
- 4주: 정규화(1NF~BCNF), 반정규화, 트랜잭션(ACID), 회복과 병행 제어
- 5주: 데이터 모델링(개념·논리·물리), ER 다이어그램, 분산 DB, 3과목 총복습 + 모의 20문제

### 4과목 프로그래밍 언어 활용 (Day 106~140)
- 1주: 프로그래밍 기초(변수·자료형·연산자), C언어 기본 문법
- 2주: 조건문·반복문, 배열과 포인터 기초, 함수
- 3주: Java 기초(클래스·객체·상속), Python 기초 문법
- 4주: 운영체제 기초(프로세스·스레드·스케줄링·메모리 관리·페이지 교체)
- 5주: 네트워크 기초(OSI 7계층·TCP/IP·IP주소·프로토콜), 4과목 총복습 + 모의 20문제
