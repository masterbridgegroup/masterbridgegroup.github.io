그 밖의 개발 모형들 (V모형·RAD·반복 점증)

2026-07-06 · 오늘 배울 개념 4개 + 퀴즈 10문제 (약 1시간)

복습 한 스푼 🥄

어제 배운 스크럼은 스프린트(1~4주 전력 질주)를 반복하는 방식이었죠. 백로그(할 일 목록) → 계획 → 스프린트 → 일일 회의 → 검토 → 회고 순서, 기억나시죠? 오늘은 시험에 종종 나오는 나머지 개발 모형들을 정리하고, 지금까지 배운 모형을 전부 비교해요.

1. V모형 (V-Model)

폭포수 모형을 V자 모양으로 접은 모형이에요. 왼쪽 내리막은 개발 단계, 오른쪽 오르막은 테스트 단계인데, 개발 단계마다 짝이 되는 테스트가 하나씩 있어요.

개발 단계 (왼쪽)↔ 짝이 되는 테스트 (오른쪽)
요구사항 분석인수 테스트 (고객이 최종 확인)
시스템 설계시스템 테스트
상세 설계통합 테스트
코딩단위 테스트
비유 — 시험 공부할 때 단원마다 확인 문제를 푸는 것처럼, 개발의 매 단계마다 대응하는 검증이 있는 거예요.
시험 포인트 — "폭포수의 변형", "테스트(검증)를 강조한 모형" = V모형.

2. RAD 모형 (빠른 개발)

RAD(Rapid Application Development)는 이름 그대로 빨리 만드는 게 목표예요. 자동화 도구(코드를 대신 만들어주는 프로그램)와 이미 만들어진 부품을 재사용해서 짧은 기간(보통 60~90일) 안에 완성해요.

비유 — 조립식 가구예요. 나무를 직접 자르지 않고 미리 가공된 부품을 조립하니 빠르죠. 대신 아주 특수한 맞춤 가구(기술적으로 복잡한 시스템)에는 안 맞아요.
시험 포인트 — "도구 활용", "60~90일", "빠른 개발" = RAD. 단, 기술적 위험이 크거나 대규모인 프로젝트에는 부적합해요.

3. 반복 점증적 모형

소프트웨어를 한 번에 다 만들지 않고, 핵심 기능부터 만들어 내놓고 조금씩 살을 붙여가는 방식이에요. 증분(增分, 늘어난 부분)을 하나씩 더해간다고 해서 '증분형'이라고도 불러요.

비유 — 게임 회사가 기본 게임을 먼저 출시하고, 업데이트로 새 기능을 계속 추가하는 것과 같아요.

애자일·나선형도 크게 보면 이 '반복' 계열에 속해요. 반대로 폭포수·V모형은 '한 번에 쭉' 계열이죠.

4. 모형 총정리 비교표

모형핵심 키워드어울리는 상황
폭포수순차 진행, 못 되돌아감요구사항이 처음부터 명확할 때
프로토타입시제품 먼저고객이 원하는 게 불분명할 때
나선형위험 분석 반복실패 위험이 큰 대형 프로젝트
V모형단계별 테스트 짝신뢰성(검증)이 중요할 때
RAD도구 활용, 60~90일빨리 만들어야 할 때
애자일짧은 반복, 변화 대응요구사항이 자주 바뀔 때
시험 포인트 — 이 표 한 장이 모형 문제의 80%를 해결해요. 키워드 → 모형 매칭을 반복해서 외우세요.
