개발 기술 환경 (OS·DBMS·미들웨어·오픈소스)

2026-07-08 · 오늘 배울 개념 4개 + 퀴즈 10문제 (약 1시간)

복습 한 스푼 🥄

어제는 현행 시스템 파악 3단계를 배웠죠. ① 구성·기능·인터페이스 → ② 아키텍처·소프트웨어 → ③ 하드웨어·네트워크, "구기인→아소→하네"! 오늘은 새 시스템을 만들 때 어떤 기술 제품을 고를지 판단하는 기준을 배워요.

1. 운영체제(OS) 선정 기준

운영체제(Operating System)는 컴퓨터를 움직이는 기본 프로그램이에요. Windows, Linux(리눅스), macOS, Android, iOS가 대표적이죠. 시스템에 쓸 OS를 고를 때 따지는 5가지:

기준뜻
신뢰도갑자기 멈추거나 오류 나지 않는가
성능동시에 많은 작업을 잘 처리하는가
기술 지원문제 생기면 도와줄 회사·커뮤니티가 있는가
주변 기기프린터 등 장비를 잘 지원하는가
구축 비용가격이 적당한가
외우는 법 — "신성기주구" (신뢰도·성능·기술지원·주변기기·구축비용). 자동차 고르기와 똑같아요: 고장 안 나고, 잘 달리고, AS 되고, 부품 많고, 가격 맞고!

2. DBMS 선정 기준

DBMS(데이터베이스 관리 시스템)는 데이터를 저장하고 관리해주는 프로그램이에요. Oracle(오라클), MySQL, PostgreSQL 등이 있어요. 선정 기준 5가지:

가용성(장애 없이 계속 쓸 수 있나) · 성능(빠른가) · 기술 지원 · 상호 호환성(다른 시스템·OS와 잘 맞나) · 구축 비용

OS 기준과 비교 — 거의 같은데 두 개가 달라요. OS는 '신뢰도·주변기기', DBMS는 '가용성·상호 호환성'. 데이터는 24시간 살아있어야 하니 가용성, 여러 프로그램과 연결되니 호환성!
시험 포인트 — OS 기준과 DBMS 기준을 바꿔치기해서 내는 문제가 나와요. 위 차이 2개만 구분하면 됩니다.

3. 미들웨어와 WAS

미들웨어(middleware)는 이름 그대로 중간(middle)에서 프로그램들을 연결해주는 소프트웨어예요. 그중 시험에 자주 나오는 게 WAS(웹 애플리케이션 서버) — 웹사이트에서 로그인, 주문 처리처럼 동적인(그때그때 달라지는) 작업을 처리하는 서버예요. Tomcat(톰캣), JEUS(제우스) 등이 있어요.

비유 — 식당의 홀 직원이에요. 손님(웹 브라우저)과 주방(데이터베이스) 사이에서 주문을 전달하고 음식을 나르죠.
시험 포인트 — WAS 선정 기준에는 특히 "안정성"이 중요해요. 그리고 "정적 페이지는 웹 서버, 동적 처리는 WAS"로 구분합니다.

4. 오픈소스 라이선스

오픈소스는 소스 코드(프로그램 설계도)가 공개되어 무료로 쓸 수 있는 소프트웨어예요. 단, 공짜라도 라이선스(사용 조건)를 지켜야 해요.

라이선스특징
GPL이걸 써서 만든 프로그램도 소스 코드를 공개해야 함 (가장 엄격)
LGPLGPL보다 완화 — 라이브러리로만 쓰면 공개 안 해도 됨
BSD / Apache자유로움 — 소스 공개 의무 없음 (저작권 표시는 필요)
시험 포인트 — "GPL은 소스 공개 의무가 있다"가 핵심. 오픈소스 선택 시 라이선스 종류·사용자 수·기술 지속 가능성을 검토해요.
