3주차 총복습 + 종합 테스트

2026-07-24 · 이번 주 핵심 정리 + 종합 테스트 15문제 (약 1시간)

복습 한 스푼 🥄

어제 배운 응집도의 결론 한 줄: 이상적인 모듈은 자료 결합도 + 기능적 응집도예요. 필요한 값만 주고받고(자료 결합), 자기 일 하나만 똑부러지게 하는(기능적 응집) 모듈이 최고의 모듈입니다. 오늘은 이번 주 전체를 정리하고 15문제로 실력을 점검해요!

1. 유스케이스 다이어그램 요약 (Day 15)

사용자 관점에서 시스템 기능을 그린 그림. 액터(막대 사람, 사각형 밖) + 유스케이스(타원) + 관계.

관계구분 키워드
포함 (include)반드시 함께 실행 (점선 화살표)
확장 (extend)조건부·선택적 실행 (점선 화살표)
일반화부모-자식 관계 (빈 삼각형 화살표)
기억! — 반드시 = include / 선택 = extend. 액터는 외부 시스템도 가능.

2. 클래스 다이어그램 요약 (Day 16)

3칸 사각형(이름 → 속성 → 오퍼레이션), 접근 제어자(+public, -private, #protected, ~package).

관계 6가지: 연관(실선) · 집합(빈 마름모, 부분 독립 가능) · 합성(찬 마름모, 운명 공동체) · 일반화(빈 삼각형+실선) · 의존(점선 화살표) · 실체화(빈 삼각형+점선)

기억! — 집합 vs 합성 = 마름모 속이 비었나 찼나. 학교-학생 vs 몸-심장.

3. 시퀀스 다이어그램 요약 (Day 17)

시간 순서(위→아래)로 객체 간 메시지를 그리는 동적 다이어그램. 생명선(존재 시간, 점선) + 활성 상자(일하는 구간) + 메시지.

동기 메시지 = 전화(기다림, 찬 화살촉) / 비동기 = 문자(안 기다림, 열린 화살촉) / 응답 = 점선.

친구들: 커뮤니케이션(관계 중심 + 메시지 번호), 상태(한 객체의 상태 변화), 활동(업무 흐름 순서도).

기억! — "번호로 순서 표현" = 커뮤니케이션, "하나의 객체의 상태 변화" = 상태, "업무 흐름 순서도" = 활동 다이어그램.

4. 설계와 모듈화 요약 (Day 18)

상위 설계(전체 뼈대·아키텍처) vs 하위 설계(모듈 내부 상세). 아키텍처 패턴: 레이어, 클라이언트-서버, 파이프-필터(컨베이어 벨트), MVC(모델·뷰·컨트롤러).

모듈화 = 레고 블록처럼 쪼개기. 추상화 = 핵심만 남기기. 정보 은닉 = 내부 감추고 인터페이스로만 소통. 팬인 = 들어오는 화살표(나를 부르는 수), 팬아웃 = 나가는 화살표(내가 부르는 수).

기억! — 좋은 모듈: 결합도 ↓ 응집도 ↑, 팬인 ↑ 팬아웃 ↓.

5. 결합도·응집도 요약 (Day 19~20)

결합도 (사이, 낮을수록 좋음)응집도 (안, 높을수록 좋음)
암기자스제외공내 (좋음→나쁨)우논시절교순기 (나쁨→좋음)
최고자료 결합도기능적 응집도
최악내용 결합도우연적 응집도

구분 키워드: 자료구조 통째 전달=스탬프 / 제어 플래그=제어 / 전역 변수=공통 / 출력→다음 입력=순차적 / 같은 데이터 사용=교환적 / 초기화 모듈=시간적.

기억! — 이 표만 통째로 외우면 이번 주 시험 문제 절반은 해결됩니다.
