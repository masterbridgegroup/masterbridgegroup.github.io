현행 시스템 파악하기

2026-07-07 · 오늘 배울 개념 4개 + 퀴즈 10문제 (약 1시간)

복습 한 스푼 🥄

어제의 핵심은 V모형 — 폭포수를 V자로 접어 개발 단계마다 테스트 짝을 붙인 모형이었죠. 요구사항 분석↔인수 테스트, 코딩↔단위 테스트. 오늘은 새 시스템을 만들기 전에 지금 쓰고 있는 시스템(현행 시스템)을 조사하는 방법을 배워요.

1. 왜 현행 시스템을 파악할까?

새 소프트웨어는 보통 기존 시스템을 대체하거나 연결돼요. 그래서 개발 전에 "지금은 뭘, 어떻게 쓰고 있지?"를 정확히 조사해야 새 시스템의 개발 범위와 방향을 정할 수 있어요.

비유 — 주방 리모델링 전에 지금 주방의 크기, 수도·가스 배관 위치, 쓰던 가전을 먼저 확인하는 것과 같아요. 안 그러면 새 싱크대가 배관과 안 맞겠죠.

2. 파악 순서 3단계

단계조사 내용
1단계시스템 구성(어떤 시스템들이 있나) / 기능(각각 뭘 하나) / 인터페이스(서로 어떻게 연결되나)
2단계아키텍처(전체 구조도) / 소프트웨어(설치된 프로그램, 라이선스)
3단계하드웨어(서버 사양) / 네트워크(연결 구성도)
외우는 법 — 위에서 아래로 내려간다고 생각하세요. ① 겉으로 보이는 기능 → ② 소프트웨어 → ③ 기계(하드웨어)와 선(네트워크). "구기인 → 아소 → 하네"
시험 포인트 — "몇 단계에서 무엇을 파악하는가"를 묻습니다. 특히 인터페이스는 1단계, 네트워크는 3단계!

3. 인터페이스 파악 시 확인할 것

인터페이스(interface)는 시스템끼리 데이터를 주고받는 접점이에요. 조사할 항목:

① 데이터 형식 — XML, JSON 등 (둘 다 데이터를 문자로 표현하는 약속된 양식)
② 전송 방식(프로토콜) — TCP/IP, HTTP 등 통신 규칙
③ 주기 — 실시간으로 보내나, 하루 한 번 모아서 보내나(배치)

비유 — 회사끼리 서류를 주고받을 때 "어떤 양식으로(형식), 어떤 방법으로(우편/이메일=프로토콜), 얼마나 자주(주기)" 보내는지 정하는 것과 같아요.

4. 소프트웨어 아키텍처란?

아키텍처(architecture)는 원래 '건축'이라는 뜻이에요. 소프트웨어 아키텍처는 시스템 전체의 뼈대 설계도 — 어떤 부품(구성 요소)들이 있고, 서로 어떻게 연결되는지를 큰 그림으로 그린 거예요.

비유 — 아파트의 골조 도면이에요. 벽지 색(세부 코드)이 아니라 기둥과 층 구조(전체 구조)를 보여주죠.
시험 포인트 — 아키텍처 = "시스템의 전체 구조", 세부 코드가 아니라 큰 그림이라는 점을 기억하세요.
